##项目文件结构
```
├─doc
│      restsql基本框架运行示意图.jpg
│      架构说明.md
│      架构说明1.1.md
│
├─restsql
│  │  query.py  -->对请求协议的封装类
│  │  restClient.py -->restsql主要的服务类，也是唯一对外暴露的接口。输入请求协议字典，输出DataFrame格式数据
│  │  __init__.py
│  │
│  ├─config     -->与数据源配置有关的文件
│  │      settings.py   --> 1.DataBase类：封装数据源信息  2.db_configs：单例类，维护一个数据源字典
│  │      table.py      --> table类：封装表信息 field类：封装字段信息
│  │      __init__.py
│  │
│  ├─datasource     -->不同数据源的查询逻辑，需要实现的核心包
│  │      druidClient.py  
│  │      esClient.py
│  │      pgClient.py   -->都继承自client类，具体需要实现的接口查看client类
│  │      __init__.py
│  │
│  └─test -->查询测试
│          druid_test.py
│          es_test.py
│          pg_test.py
│          __init__.py
│
└─RestSQLServer -->服务端读取配置文件，在db_configs中注册数据源对象。
                    通过拦截URL，并调用相应的视图层来完成逻辑。
                    完成与前端的交互工作。
                    具体文件结构，待完善restsql包后进行具体规划
```

##模块划分

###restsql
```
1.esclient实现 
    elasticsearch-DSL实现
2.pgclient实现  
    考虑到sql注入安全问题，两种解决方案
     orm 或者 占位符传参方式（类似java PrepareStatement）
3.druidclient实现 
    拼接sql方式实现
项目的核心部分：尽可能多的测试，保证没有bug
```
###restSqlServer
```
1.接口的定义与实现
    例如：grafana插件中实现 "/","/query","/search","/find_options","/find_tables"
    接口遵循RESTful规范，提供充分的服务
2.配置文件的读取
    将配置文件中的数据源信息实例化为DataBase对象，并在db_config中添加注册
3.简单前端的实现
    为了演示效果，可以考虑实现简单的表单页面或者grafana插件开发展示
  以及通过matplotlib实现数据的可视化
   
    

```
